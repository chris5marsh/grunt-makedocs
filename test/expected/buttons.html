Lots of Buttons

Buttons come in four different types and three different sizes. Usually a button element should be used, although an a, an input[type=button], an input[type=reset] or an input[type=submit] could be used.

The simplest is just a plain button:

<input type="submit" class="btn" value="Submit">

Other types are cancel, primary and highlight:

<button class="btn btn-highlight">Or this</button>

You can define the size of the button too:

<button class="btn btn-huge">Huge button</button>

It's also easy to add an icon to a button, either at the front, the end, or both (which doesn't look good, so don't do it).

<button class="btn"><i class="icon-mobile"></i>Please don't<i class="icon-tick"></i>
</button>

You can join this all together to make a monster button:

<button class="btn btn-warning btn-large">Oh gosh<i class="icon-tick"></i>
</button>

Button groups

You can combine buttons together in a btn-group. This will join the buttons together seamlessly.

<div class="btn-group">
  <button class="btn">Back</button>
  <button class="btn btn-warning">Help<i class="icon-help"></i>
  </button>
  <button class="btn">Next</button>
</div>

Complete options

Component names

  • button
  • button-link
  • button-input
  • button-submit

Options

Required

  • text: the text on the button

Optional

  • type: one of default (default), cancel, primary and highlight
  • size: one of tiny, small, medium (default), large or huge
  • icon-before: the type of icon to appear at the front of the button (doesn't work with input[type=submit] or input[type=button])
  • icon-after: the type of icon to appear at the end of the button (doesn't work with input[type=submit] or input[type=button])
  • href: the URL to visit when clicked (only for button-link, defaults to "#")
